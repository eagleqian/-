
功能概要：构件化理解UART编程

版本更新：2016-03-14 V4.0 (更新到CMSIS标准)
硬件说明：PORTB19=红灯；PORTB18= 绿灯；PORTB9= 蓝灯；

1. 工程概述
(1)本工程为KL25微控制器无操作系统应用工程，ADC样例程序
(2)使用串口与外界通信，使用串口1，波特率为9600，1位停止位，无校验

2. 软件环境
(1)Kinetis Design Studio IDE v3.0.0.

3. 硬件环境
(1)KL25 SD13-01 EVB板
(2)蓝灯对应引脚为PTB9,红灯对应引脚为PTB19，绿灯对应引脚为PTB18
(3)串口发送引脚为PTE0，接收引脚为PTE1
(4)ADC0通道0为引脚PTE20，连接外部电压0~3.3V
(5)ADC0通道3为引脚PTE22，连接外部电压0~3.3V
(6)ADC0通道4a为引脚PTE21，连接外部电压0~3.3V
(7)ADC0通道7a为引脚PTE23，连接外部电压0~3.3V
(8)ADC0通道4b为引脚PTE29，连接外部电压0~3.3V
(9)ADC0通道5b为引脚PTD1，连接外部电压0~3.3V
(10)ADC0通道6b为引脚PTD5，连接外部电压0~3.3V
(11)ADC0通道7b为引脚PTD6，连接外部电压0~3.3V
(12)ADC0通道11为引脚PTC2，连接外部电压0~3.3V
(13)ADC0通道12为引脚PTB2，连接外部电压0~3.3V
(14)ADC0通道13为引脚PTB3，连接外部电压0~3.3V
(15)ADC0通道14为引脚PTC0，连接外部电压0~3.3V
(16)ADC0通道15为引脚PTC1，连接外部电压0~3.3V
(17)ADC0通道23为引脚PTE30，连接外部电压0~3.3V
    
4. 实验现象
(1)程序启动时串口发送This is ADC Test！、校验成功！，同时灯闪烁
(2)蓝灯闪烁
(3)根据各个引脚的的输入电压，串口输出范围00_00 ~ 03_FF
(4)初始化ADC模块时，选择的是a通道组，然后对各个通道都进行一次采样，并将采样的结果存入数组中，
        用串口将数组的数据发送给PC端，可以在PC看到采集的数据，用杜邦线将一端接地，一端接通道对应的引脚时，
        可以看到相应的通道采集数据变为0。   

5. 实验注意
(1)可以通过高端PC c#2010及以上的版本来实时接受数据
(2)先打开串口，后开启KL25 SD13-01 EVB板电源
(3)C#2010串口测试程序中包含十进制数据发送和十六进制数据发送
