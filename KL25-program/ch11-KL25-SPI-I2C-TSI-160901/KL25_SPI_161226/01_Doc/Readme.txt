硬件说明：KL25的SPI0和SPI1通信。
	   SPI主从机模块间的通信，必须要在主机的时钟信号的控制下才得以进行；
	        主机模块在发送数据时会同时产生时钟信号，因此当从机向主机发送数据时，
	        主机仍要有数据发送以产生时钟控制信号（发送的数据应被从机忽略）。
	        否则则会导致从机发送数据失败。


1. 工程概述
(1)本工程为KL25微控制器无操作系统应用工程，SPI样例程序
(2)使用串口与外界通信：
        主机(SPI_0)使用串口1，波特率为9600；
        从机(SPI_1)使用串口2，波特率为9600；
(3)启动串口接收中断，回发接收数据
(4)初始化SPI_0和SPI_1。
(5)进入循环SPI_0向SPI_1发送字符单个A~Z并回显；
	SPI_1通过中断接收到这个字符并判断，通过串口回显并将接收到的数据“+1”后返回给SPI_0。


2. 软件环境
(1)Kinetis Design Studio IDE v3.0.

3. 硬件环境
(1)KL25 SD13-01 EVB板（两块板）
(2)蓝灯对应引脚为PTB9
(3)SPI0对应的引脚为PTA14、PTA15、PTA16、PTA17其分别连接SPI1的PTB10、PTB11、PTB16、PTB17引脚。
   （可参照spi.c文件中的初始化定义引脚）
4. 实验现象
随着蓝灯亮暗交替变化，主从机串口不停的输出A~Z,复位之后输出This is a SPI test!

注意：读者若需其他的波特率，可自行查看寄存器进行配置
